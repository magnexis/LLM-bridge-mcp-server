# Jobs And Workflows

Jobs represent persisted orchestration or workflow planning state.

Workflows are reusable definitions that can be turned into jobs through `glm_5_run_workflow`.

Built-in workflows currently include:

- `repository_audit`
- `fix_failing_tests`
- `safe_refactor`
- `dependency_upgrade`
- `security_remediation`
- `release_readiness`
- `ui_quality_review`
- `documentation_refresh`
